# Product Requirements Document

## 1. Problem Statement
Пользователи не знают точно свой тип кожи и испытывают трудности при выборе средств по уходу. Часто советы в интернете противоречивы, а косметические бренды дают универсальные рекомендации, которые не подходят конкретному человеку.

## 2. Target Audience
* **Возраст:** 18-35 лет.
* **Пол:** любой.
* **Интересы:** уход за кожей, косметика, self-care, технологии.
* **Поведение:** ищут советы онлайн, покупают косметику в интернете.

## 3. Value Proposition
* **Персонализация:** рекомендации формируются на основе реального фото лица и типа кожи.
* **Простота:** интерфейс для загрузки фото и получения результата менее чем за минуту.
* **Экспертность:** подбор ингредиентов (ниацинамид, кислоты и др.) на основе дерматологических протоколов.
* **История:** сохранение истории анализов для наблюдения за динамикой.

## 4. MVP Scope
* **Регистрация и профиль**: создание аккаунта для хранения данных.
* **Загрузка фото**: отправка фото для анализа нейросетью.
* **Диагностика**: автоматическое определение типа кожи.
* **Рекомендации**: получение персонального списка рекомендаций по уходу.
* **История анализов**: сохранение результатов для отслеживания динамики состояния лица.

## 5. User Stories
* Как пользователь, я хочу загрузить фото лица, чтобы узнать свой тип кожи и получить персональные рекомендации.
* Как пользователь, я хочу видеть историю моих анализов, чтобы отслеживать изменения состояния кожи во времени.
* Как пользователь, я хочу получить рекомендации по ингредиентам и уходовым средствам, чтобы покупать подходящие продукты и не тратить деньги на неподходящие.

## 6. Non-Functional Requirements
* **Время отклика** API < 200 мс.
* **Нагрузка**: поддержка до 100 RPS.
* **Безопасность**: аутентификация, валидация данных.
