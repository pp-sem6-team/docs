# Регламент разработки

## 1. Ветки
* Все ветки создаются от `dev`.
* Прямые коммиты в `main` и `dev` **запрещены**.
* **Формат названия ветки:** `тип/описание-ИДЕНТИФИКАТОР_ЗАДАЧИ`
  * `feat/` — новый функционал.
  * `fix/` — исправление багов.
  * `docs/` — документация.

**Пример:** `feat/update-models-BACKEND-11`

## 2. Коммиты
Придерживаемся **Conventional Commits** (`[ИДЕНТИФИКАТОР_ЗАДАЧИ] тип(область): описание`):
* `feat` — новый функционал.
* `fix` — исправление ошибки.
* `docs` — работа с документацией.
* `refactor` — улучшение кода без изменения логики.

**Пример:** `[BACKEND-12] feat(db): add database models`

## 3. Pull Requests
* **Заголовок:** `ИДЕНТИФИКАТОР_ЗАДАЧИ: Краткое описание` (например, `FRONTEND-13: Add login form`).
* **Описание:** Желательно указывать `Closes: #НОМЕР_ЗАДАЧИ` (например, `Closes: #13`).
* **Слияние изменений:** Минимум 1 аппрув для `dev`, 2 аппрува для `main`.
